## Reusable technique (important)
In this sandbox the managed venv had NO numpy/matplotlib/openpyxl and `pip install` was extremely slow. Worked around it with **pure stdlib only**:
- Monte Carlo → `random.Random` + `statistics`, not numpy.
- Charts → hand-built **SVG** strings (line/bar helpers), embedded inline in HTML (no matplotlib, fully self-contained).
- Excel → minimal **xlsx via `zipfile` + inline-string cells** (no openpyxl). OOXML skeleton: `[Content_Types].xml`, `_rels/.rels`, `xl/workbook.xml`, `xl/_rels/workbook.xml.rels`, `xl/worksheets/sheetN.xml`.
This is the fallback whenever heavy data/viz libs aren't installable. Consider a skill for "stdlib-only finance/deliverable generation".
